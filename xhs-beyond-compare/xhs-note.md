# 小红书标题
用 Claude 搞了个「Beyond Compare 替代技能」，帮团队一年省下几百刀授权费

## 正文

最近用 Claude 给自己撸了一个小技能。

场景你肯定熟：
- 想用 Beyond Compare 看代码 diff，体验确实好
- 但授权按美元算，一个人几十美刀，小团队一年就是几百刀
- 大部分时间只是偶尔用一下，买了心里又有点不甘心

所以我干脆直接在 Claude 里搞了一个「beyond-compare」技能：
在 Claude Code 里敲一行 `/beyond-compare`，
它会自动把这次代码改动分析出来，生成一份 Beyond Compare 风格的 HTML 报告。

从读者（也就是你）角度，这个技能能帮你解决几个现实问题：

1⃣ 不再被丑陋的 git diff 折磨
- 左右并排对比：左边旧版本，右边新版本
- 红色删除、绿色新增，一眼看出到底改了什么
- 顶部统计卡片：改了多少文件、+多少行、-多少行、净变化多少

2⃣ 更适合做 code review 和复盘
- 生成的是一个独立 HTML 文件，直接浏览器打开
- 可以投屏讲解，也可以截图放到周报 / 复盘文档里
- 发给产品 / 测试 / 运营看，也能看懂，不需要他们装任何工具

3⃣ 真正帮你省钱
- 工具本身是开源的，用的是 Claude 的技能机制，不需要再买 Beyond Compare 授权
- 如果你原来打算给团队买几套 Beyond Compare，这个技能至少能帮你省下几百刀预算
- 这些钱可以用在更直接提升自己的地方：课程、书、训练营，而不是一个偶尔才用的 diff 工具

我自己的使用场景：

- 大改动前给自己看一眼，确认没有漏改、不乱改
- 做 code review 时，直接打开 HTML 给同事讲解，比在终端里翻 diff 友好太多
- 写「性能优化复盘」「大重构总结」的时候，直接截图这个页面，当作图文说明

项目我已经开源了，代码在这：
👉 https://github.com/mason0510/beyond-compare

如果你：
- 一直想用 Beyond Compare，但又不想为偶尔使用每年多付几百刀
- 经常需要跟别人解释「这次代码到底改了什么」
- 想把自己的重构/优化做成「可视化报告」展示出去

那不妨先把这个技能用起来，把钱留给更值得投资的地方。
